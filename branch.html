Rangkuman Materi Git Branch

Pengertian Branch

Branch merupakan cabang dari repository untuk memanage pengembangan dalam GIT.

Perintah Branch

  1. Git branch: melihat semua branch (local)
  2. Git branch --all: melihat semua branch (public)
  3. Git branch nama_branch: membuat branch baru
  4. Git checkout nama_branch: untuk berpindah branch
  5. Git branch -d nama_branch: untuk menghapus branch
  6. Git merge nama_branch
    • untuk menggabungkan commit dari 2 branch
    • Relatif terhadap branch yang aktif saat ini
    • Untuk melakukan merge harus dipastikan tidak ada file yang belum di commit
  7. Conflict
    • Kondisi yang terjadi ketika merging 2 branch yang memiliki perubahan pada file yang sama
    • Ketika conflict, kita tidak bisa melakukan commit
    • Untuk melakukan commit, conflict harus diselesaikan dulu
